프로그래밍 언어의 `함수`는 **일련의 과정을 문으로 구현하고 코드 블록으로 감싸서 하나의 실행 단위로 정의한 것이다.**

입력 값을 받아서 출력 값을 내보낸다. 이때 입력을 전달받는 변수를 **매개변수**, 입력을 **인수**, 출력을 **반환 값**이라 한다.

함수 명시적으로 호출해야 사용할 수 있는데 이를 함수 호출이라 한다.

```jsx
funtion add(x,y) {
	return x + y
}

// 함수 호출
var result = add(2,5);

console.log(result) /// 7
```

# 1. 함수

## 1.1 함수를 사용하는 이유

---

함수는 필요할 때 여러번 호출할 수 있다. 즉, **실행 시점을 개발자가 결정할 수 있고 몇 번이든 재사용이 가능하다.**

잘 만든 함수는 **유지보수의 편의성**을 높이고 **코드의 신뢰성**을 높이는 효과가 있다.

함수는 객체 타입 값이다. 따라서 식별자(이름)을 붙일 수 있다. 함수 이름은 변수 이름과 마찬가지로 함수의 역할을 적절히 잘 설명할 수 있도록 지어야한다. 이는 **코드의 가독성**을 향상 시킨다.

## 1.2 함수 리터럴

---

자바스크립트의 함수는 객체 타입의 값이다. 따라서 다른 리터럴의 값 처럼 생성해 변수에 할당할 수 있다. 함수 리터럴은 function 키워드, 함수 이름, 매개변수, 함수 몸체로 구성된다.

```jsx
// 변수에 함수 리터럴을 할당
var func = function add(x, y) {
  return x + y;
};
```

리터럴은 사람이 이해할 수 있는 문자 또는 약속된 기호에 사용해 값을 생성하는 표기 방식을 말한다. 즉, 리터럴은 값을 생성하기 위한 표기법이다. 따라서 리터럴도 평가되어 값을 생성하며 이 값은 객체다.

즉, **함수는 객체 이다.**

## 1.3 함수의 정의

---

정의한 함수를 호출하기 이전에 인수를 전달받을 매개변수와 실행할 문들, 그리고 반환할 값을 지정하는 것을 말한다.

| 방식명                             | 설명                                                                            | 예시                                            |
| ---------------------------------- | ------------------------------------------------------------------------------- | ----------------------------------------------- |
| 함수 선언문 (Function Declaration) | 함수 이름을 명시적으로 선언하며, 호이스팅이 적용되어 함수 정의 전에도 호출 가능 | `function sum(a, b) { return a + b; }`          |
| 함수 표현식 (Function Expression)  | 변수에 함수를 할당하는 형태로, 함수 선언 후에 호출 가능                         | `const sum = function(a, b) { return a + b; };` |
| 화살표 함수 (Arrow Function)       | ES6에서 도입된, function 키워드 없이 간결하게 함수를 정의할 수 있는 방식        | `const sum = (a, b) => a + b;`                  |

### 1.3.1 함수 선언문

함수 선언문을 사용해 함수를 정의하는 방식

```jsx
// 함수 선언문
function add(x, y) {
  return x + y;
}

// 함수 참조
console.dir(add); // f.add(x,y)
// 함수 호출
console.log(add(2, 5)); // 7
```

함수 선언문은 함수 리터럴과 형태가 동일하다. 단. **함수 리터럴은 함수 이름을 생략할 수 있지만 선언문은 함수 이름을 생략할 수 없다.**

**함수 선언문은 표현식이 아닌 문이다.**

표현식이 아닌 문은 변수에 할당할 수 없다. 함수 선언문도 표현식이 아닌 문이므로 변수에 할당할 수 없다. 하지만 예외가 있다.

```jsx
var add = function add(x, y) {
  return x + y;
};

// 함수 호출
console.log(add(2, 5)); // 7
```

위 예제는 함수 선언문이 변수에 할당 되는 것처럼 보이는데, 이 코드가 동작하는 이유는 자바스크립트 엔진이 코드의 문맥에 따라 동일한 함수 리터럴을 표현식이 아닌 문인 `함수 선언문`으로 해석하는 경우와 표현식인 문인 `리터럴 표현식`으로 해석하는 경우가 있기 때문이다.

피연산자가 존재한다면 함수 표현식으로 해석되고 단독으로 존재한다면 함수 선언문으로 해석된다.

```jsx
// 기명 함수 리터럴을 단독으로 사용하면 함수 선언문으로 해석된다.
// 함수 선언문에서는 함수 이름을 생략할 수 없다.
function foo() {
  console.log("hi");
}
foo(); // hi

// 함수 리터럴을 피연산자로 사용하면 함수 선언문이 아니라 함수 리터럴 표현식으로 해석된다.
(function bar() {
  console.log("bar");
});
bar(); // ReferenceError: bar is not defined
```

함수 이름은 함수 몸체 내에서만 참조할 수 있는 식별자다. 이는 함수 몸체 외부에서는 함수 이름으로 함수를 참조할 수 없으므로 함수 몸체 외부에서는 함수 이름으로 함수를 호출 할 수 없다는 의미이다.

**즉, 함수를 가리키는 식별자가 없다는 것과 마찬가지다.**

그러면 함수 선언문은 어떻게 호출해서 사용할 수 있는 것일까? 그건 **자바스크립트 엔진에서 함수를 호출하기 위해 함수 이름과 동일한 이름의 식별자를 생성하고 거기에 함수 객체를 할당하기 때문이다.**

함수 선언문을 함수 표현식으로 변환해 함수 객체를 생성한다고 생각할 수 있는데 **함수 선언문과 함수 표현식이 정확히 같은 동작을 하지는 않는다.**

### 1.3.2 함수 표현식

값의 성질을 갖는 객체를 일급 객체라 한다. 자바스크립트의 함수는 `일급 객체`다. 함수가 `일급 객체` 라는 것은 함수를 값처럼 자유롭게 사용할 수 있다는 의미다.

함수는 일급 객체 이기 때문에 함수 리터럴로 생성한 함수 객체를 변수에 할당할 수 있다.

이를 `함수 표현식` 이라고 한다.

```jsx
var add = function (x, y) {
  return x + y;
};
console.log(add(2, 5)); // 7
```

함수 리터럴은 함수 이름을 생략할 수 있는데 이러한 함수를 익명 함수라고 한다.

```jsx
// 기명 함수 표현식
var add = function foo(x, y) {
  return x + y;
};
// 함수 객체를 가리키는 식별자 호출
console.log(add(2, 5)); // 7

// 함수 이름으로 호출하면 참조 에러 발생
// 함수 이름은 몸체 내부에서만 유효하다.
console.log(foo(2, 5)); // ReferenceError: foo is not defined
```

<aside>
📔 함수 선언문은 표현식이 아닌 문이고 함수 표현식은 표현식인 문이다.

</aside>

### 1.3.3 함수 생성 시점과 함수 호이스팅

```jsx
console.log(add); // f add(x, y)
console.log(sub); // undefined

console.log(add(2, 5)); // 7
console.log(sub(2, 5)); // TypeError: sub is not a function

function add(x, y) {
  return x + y;
}

var sub = function (x, y) {
  return x + y;
};
```

함수 선언문으로 정의한 함수와 표현식으로 정의한 함수의 생성 시점은 다르다.

`함수 선언문`은 코드가 실행되는 시점인 런타임 이전에 자바스크립트 엔진에 의해 먼저 실행되어 객체가 생성된다. 그리고 **암묵적으로 함수 이름과 동일한 식별자를 생성하고 생성된 함수 객체를 생성한다. 이를 이용해서 함수 선언문이 코드의 선두로 끌어 올진 것처럼 동작하는 자바스크립트 고유의 특징을 함수 호이스팅 이라한다.**

`함수 표현식`에서는 변수에 할당되는 것은 함수 리터럴이다. 그렇기 때문에 동작하는 방식이 var 키워드를 사용한 방식의 할당과 다르지 않다. 즉, **변수 할당문의 값은 할당문이 실행되는 시점 다시말해 런타임에 평가되므로 함수 표현식의 함수 리터럴도 할당문이 실행되는 시점에 평가되어 함수 객체가 된다.**

따라서 **함수 표현식으로 함수를 정의하면 함수 호이스팅이 발생하는 것이 아니라 변수 호이스팅이 발생한다.**

### 1.3.4 Function 생성자 함수

- 생성자 함수 (constuctor function)

  생성자 함수는 객체를 생성하는 함수를 말한다. 객체를 생성하는 방식은 객체 리터럴 이외에 다양한 방법이 있다.

```jsx
var addd = new Function("x", "y", "return x + y");

console.log(add(2, 5)); // 7
```

Function 생성자로 만드는 것은 바람직하지 않으며 일반적이지도 않다. 클로저를 생성하지도 않으며 선언문이나 표현식으로 생성한 함수와 다르게 동작한다.

```jsx
var add1 = (function () {
	var a = 10;
	return funtion x + y + a;
}());

console.log(add1(1, 2)) //13

var add1 = (function (x, y) {
  var a = 10;
  return new Function("x", "y", "return x + y + a");
})();
console.log(add1(1, 2)); // ReferenceError: a is not defined 다르게 동작
```

### 1.3.6 화살표 함수

ES6에서 도입된 화살표 함수로 function 키워드 대신 화살표흫 사용해 좀 더 간략한 방법으로 선언

화살표 함수는 항상 익명 함수로 정의한다.

```jsx
const add = (x, y) => x + y;
console.log(add(2, 5)); // 7
```

화살표 함수는 선언문 함수와 표현식 함수를 대체하려고 나온 것이 아니다. 화살표 함수는 생성자 함수로 사용할 수 없으며, 기존 함수와 this 바인딩 방식도 다르고, prototype프로퍼티가 없으며 arguments 객체를 생성하지 않음

## 1.4 함수호출

---

함수를 실행하기 위해서는 필요한 값을 함수 외부에서 함수 내부로 전달할 필요가 있다. 이를 `매개변수`라 하고 매개변수를 통해 `인수`를 전달한다. 인수는 `표현식` 이어야하며 **개수와 타입에 제한이 없다.**

```jsx
function add(x, y) {
  return x + y;
}
// 함수 호출, 인수 1과 2가 매개변수 x,y로 순서대로 할당되고 몸체 문들이 실행
```

매개변수는 함수를 정의할 때 선언하며, 함수 몸체 내부에서 변수와 동일하게 취급된다. 즉, 함수가 호출되면 함수 몸체 내에서 암묵적으로 매개변수가 생성되고 일반 변수와 마찬가지로 undefined로 초기화 된 이후 수가 순서대로 할당된다.

- 매개변수 동작 방식

  함수 호출 → 함수 몸체 내부에서 매개변수 생성 → 매개변수 undefined로 초기화 → 순서대로 할당

외부에서 함수 내부 매개변수 호출시

```jsx
function add(x, y) {
  return x + y;
}

add(2, 5);
// 매개변수 x, y는 함수 몸체 내붸서만 참조가능
console.log(x, y); // ReferenceError: x is not defined
```

매개변수가 하나 부족 시

- 매개변수가 undefined로 초기화되기 때문에 2 와 undefined의 계산이되어 NaN이 발생한다.

```jsx
function add(x, y) {
  return x + y;
}

console.log(add(2)); // NaN
```

매개변수가 초과시

- 무시해도 상관없다(모든 인수가 암묵적으로 arguments 객체의 프로퍼티로 보관된다.)

```jsx
function add(x, y) {
  console.log(arguments); // Arguments(3) [2,3,4,5, callee:f, Symbol(Symbol.iterator): f]
  return x + y;
}
console.log(add(2, 3, 4, 5)); // 5
```

### 1.4.1 인수 확인

1. 자바스크립트 함수는 매개변수와 인수 개수가 일치하는지 확인하지 않는다.
2. 자바스크립트는 동적 타입 언어다. 따라서 자바스크립트 함수는 매개변수의 타입을 사전에 지정할 수 없다.

### 1.4.2 매개변수의 최대 개수

ECMAScript 사양에서는 매개변수 최대 개수에 대해 명시적으로 제한하고 있지 않다. 하지만 매개변수는 코드를 이해하는 데 방해되는 요소이기 때문에 적을 수록 좋다. 매개변수의 개수가 많다는 것은 함수가 여러가지 일을 한다는 것이기 떄문에 바람직하지 않다. **이상적인 함수는 한 가지 일만 해야하고 가급적 작게 만들어져야 한다.**

3개가 넘어갈 시 하나의 매개변수로 선언하고 객체를 인수로 전달하는 것이 유리하다.

```jsx
$.ajax({
  method: "post",
  url: "/user",
  data: { id: 1, name: "Song" },
});
```

하지만 주의할 점은 외부에서 내부로 전달한 객체는 부수 효과가 발생한다는 것이다.

### 1.4.3 반환문 (return)

반환문은 다양한 특징이 있다.

1. 반환문은 함수의 실행을 중단하고 함수 몸체를 빠져나간다.
2. 반환문은 return 키워드 뒤에 오는 표현식을 평가해 반환한다. return 키워드 뒤에 반환 값으로 사용해야할 표현식을 명시적으로 지정하지 않으면 undefined가 반환된다.
3. 반환문은 생략할 수 있다. 이때 함수는 함수 몸체 마지막 문까지 다 실행한 후 암묵적을 undefined를 반환한다.
4. 반환문은 함수 몸체 내부에서만 사용할 수 있다.

## 1.5 참조에 의한 전달과 외부 상태의 변경

---

객체는 `참조에 의한 전달` 방식으로 동작한다. 매개변수 또한 함수 몸체 내부에서 변수와 동일하게 취급됨으로 매개변수 또한 타입에 따라 값에 의한 전달, 참조에 의한 전달 방식을 그대로 따른다.

```jsx
function changVal(primitive, obj) {
  primitive += 100;
  obj.name = "Lim";
}

var num = 100;
var person = { name: "Arm" };

console.log(num); // 100
console.log(person); // { name: Arm }

changVal(num, person);

// 원시 값은 훼손되지 않음
console.log(num); // 100
// 객체 값은 원본이 훼손된다..
console.log(person); // { name : Lim }
```

changVal 함수에서 obj의 값은 원시 값이기 때문에 재할당을 통해서 값이 변경된다. obj는 변경 불가능한 데이터이기 때문이다. person 값은 객체 값이기 때문에서 함수 몸체에서 값을 변경하면 원본 값이 훼손된다**.** 다시 말해 **원시 값인 obj에서는 원본 값이 변경되는 부수효과가 일어나지 않고 객체 값인 person은 원본 데이터가 훼손되는 부수 효과가 일어난다.**

함수가 외부 상태를 변경하면 상태 변화를 추적하기 어려워진다. 이는 코드의 복잡성을 증가시키고 가독성을 해치는 원인이 된다.

문제에 대한 해결방법은 객체를 불변 객체로 만들어 사용하는 것이고 객체의 복사본을 새롭게 생성하는 비용이 들지만 객체를 마치 원시 값처럼 변경 불가능한 값으로 동작하게 만드는 것이다. 이를 이용해 깊은 복사를 통해 새로운 객체를 생성하고 재할당을 통해 교체하면 부수 효과를 없앨 수 있다.
