# 1. 전역 변수

전역 변수를 무분별하게 사용하면 변수에 의도치 않은 값이 닮길 수 있기 때문에 위험하다. 전역 변수를 사용해야 할 이유를 찾지 못했다면 쓰지 않는 것이 좋다.

## 1.1 변수의 생명주기

---

### 1.1.1 지역 변수의 생명 주기

변수는 자신이 선언된 위치에서 생성되고 소멸한다. `전역 변수`의 생명 주기는 애플리케이션의 생명 주기와 같다. 하지만 함수 내부에서 선언된 `지역 변수`는 함수가 호출되면 생성되고 함수가 종료하면 소멸한다.

```jsx
function foo() {
  var x = "local";
  console.log(x); // local
  return;
}
foo();
console.log(x); // ReferenceError: x is not defined
```

`지역 변수` x는 foo 함수가 호출되면 생성된다. 호출되지 않으면 함수 내부의 변수 선언문이 실행되지 않기 때문이다.

변수 선언문은 선언문이 어디에 있든 상관없이 가장먼저 실행된다. 즉, 런타임 이전 단계에서 자바스크립트 엔진에 의해 선언되고 undefined로 초기화 된다. 이 **동작은 엄밀히 말하자면 전역 변수에만 해당되는 말이다.**

위 코드에서 지역 변수 x는 결국 foo 함수가 호출되었을때만 유효하다. 즉, **지역 변수의 생명 주기는 함수의 생명 주기와 일치한다.**

함수 몸체 내부에서 선언된 지역 변수의 생명 주기는 함수의 생명 주기와 대부분 일치하지만 지역 변수가 함수보다 오래 생존하는 경우도 있다.

`변수의 생명 주기`는 **메모리 공간이 확보된 시점부터 메모리 공간이 해제되어 가용 메모리 풀에 반환되는 시점이다.**

함수 내부에서 선언된 지역 변수는 함수가 생성한 스코프에 등록된다. 함수가 생성한 스코프는 렉시컬 환경이라 부르는 물리적인 실체가 있다고 했다. 따라서 변수는 자신이 등록된 스코프가 소멸될 때까지 유효하다.

하지만 **누군가 스코프를 참조하고 있으면 스코프는 소멸되지 않고 생존하게 된다.**

## 1.2 변수 호이스팅

```jsx
var x = "global";
function foo() {
  console.log(x); // (1)
  var x = "local";
}
foo();
console.log(x); // global
```

foo 함수 내부에서 선언된 지역 변수는 런타임 중 (1) 시점에서 선언되고 undefined로 초기화 된 상태이다. foo 함수의 스코프에서 지역 변수 x에 할당 되기 전에 x를 참조했기 때문에 (1)의 값은 undefined가 된다. 이때 전역변수의 값을 참조하지 않는 이유는 **호이스팅은 스코프 단위로 동작하기 때문이다.**

런타임 이전에 변수는 선언되고 undefined로 초기화된 상태이다. 이때 스코프 내에 변수가 있다면 외부에서 찾지 않고 내부 변수를 사용한다.

<aside>
💡 호이스팅은 변수 선언이 스코프의 선두로 끌어 올려진 것처럼 동작하는 자바스크립트의 특징이다.

</aside>

## 1.3 스코프 체인과 스코프 단계의 변수 참조

`스코프 체인`에 의해서 동작하게 되는데 foo 함수에서 `렉시컬 환경`을 확인해 원하는 변수가 없을 시 외부 렉시컬 환경을 확인하게 된다. 이렇게 전역 렉시컬 환경까지 반복하게 된다. 각 스코프에서 동작할때 환경 레코드를 검색해 확인하고 원하는 변수가 존재할 시 종료한다. 이렇게 반복해 전역까지 확인해도 찾을 수 없다면 에러가 발생한다.

## 1.4 전역 변수의 생명 주기

전역 변수는 코드가 로드 되자마자 곧바로 해석되고 실행된다. 함수는 몸체에 반환문이 존재할 시 반환문이 실행되면 종료된다. 하지만 전역 코드에는 반환문을 사용할 수 없기 때문에 더 이상 실행할 문이 없을 시 종료한다.

var 키워드로 선언한 전역 변수는 전역 객체의 프로퍼티가 된다. 이는 전역 변수의 생명 주기가 전역 객체의 생명 주기와 일치한다는 것을 말한다.

### 1.4.1 전역 객체에 대한 설명

전역 객체(Global Object)는 자바스크립트 환경에서 언제나 접근 가능한 객체로, 모든 전역 변수와 전역 함수, 자바스크립트 내장 객체를 프로퍼티로 포함합니다. 전역 객체는 자바스크립트 실행 환경에 따라 다른 이름과 형태를 가질 수 있습니다.

### 1.4.2 브라우저 환경에서의 전역 객체

브라우저 환경에서의 전역 객체는 `window` 객체입니다. `window` 객체는 브라우저 창이나 탭의 전역 스코프를 나타냅니다. 이 객체를 통해 DOM(Document Object Model) 조작, BOM(Browser Object Model) 관련 작업 등 브라우저와 상호 작용하는 다양한 기능을 수행할 수 있습니다. 또한, `alert`, `setTimeout`, `setInterval` 등의 전역 함수도 `window` 객체의 메서드로 제공됩니다.

### 1.4.3 Node.js 환경에서의 전역 객체

Node.js 환경에서는 `global` 객체가 전역 객체의 역할을 합니다. 이 객체는 Node.js 환경 전반에서 사용할 수 있는 전역 변수와 함수를 포함하고 있습니다. 예를 들어, `global` 객체를 통해 어느 모듈에서나 접근 가능한 변수를 정의할 수 있습니다. 그러나 전역 변수의 사용은 가급적 제한하는 것이 좋으며, 모듈 시스템을 통한 명시적인 의존성 관리가 권장됩니다.

## 1.5 ECMAScript 2015(ES6) 이후의 변경 사항

ECMAScript 2015(ES6) 이후에는 `globalThis`라는 새로운 전역 객체가 도입되었습니다. `globalThis`는 다양한 자바스크립트 환경에서 일관된 전역 객체에 접근할 수 있는 표준화된 방법을 제공합니다. 즉, 브라우저 환경에서는 `window`, Node.js 환경에서는 `global`, Web Worker 환경에서는 `self`에 해당하는 객체에 대해 `globalThis`를 사용하여 일관된 방식으로 접근할 수 있습니다.

## 1.6 전역 객체의 사용

전역 객체는 자바스크립트에서 가장 기본이 되는 객체로, 전역 스코프에 있는 변수와 함수를 포함합니다. 전역 변수나 전역 함수를 선언하면 사실상 전역 객체의 프로퍼티로 추가되는 것입니다. 그러나 전역 스코프를 과도하게 사용하는 것은 이름 충돌, 코드 관리 어려움 등의 문제를 야기할 수 있으므로, 모듈화와 지역 스코프를 활용하는 것이 좋습니다.

![전역 변수 이미지](https://raw.githubusercontent.com/thdtjdgml415/next-blog/main/assets/img/global-variable1.webp)

브라우저 환경에서 전역 객체는 window 이므로 브라우저 환경에서 var 키워드로 선언한 전역 변수는 전역 객체 window의 프로퍼티다. 따라서 **전역 객체 window는 웹페이지를 닫기 전까지 유효하다.**

### 1.6.1 전역 변수의 문제점

---

- **압묵적 결합**

  모든 코드가 전역 변수를 참조하고 변경할 수 있는 `압묵적 결합`을 허용하는 것이다. 이는 코드의 가독성을 해치고 의도치 않게 변경될 수 있기 때문에 좋지 않다.

- **긴 생명 주기**

  전역 변수의 생명 주기는 길다. **메모리 리소스도 오랜기간 소모**한다. var 키워드는 변수의 중복 선언을 허용하기 때문에 **의도치 않는 재할당**이 이뤄질 수 있다.

- **스코프 체인 상에서 종점에 존재**

  변수는 찾을 때 지역 변수에서 부터 검색하기 때문에 가장 마지막에 검색되는 것이 전역 변수다. 그렇기 때문에 **전역 변수의 검색 속도가 가장 느리다.** 차이는 많이나지 않지만 분명 차이가 존재한다.

- **네임스페이스 오염**

  자바스크립트의 가장 큰 문제점 중 하나는 **파일이 분리되어 있어도 전역 변수를 공유하는 것**이다. 이는 동일한 이름의 변수를 선언하면 **예기치 못한 할당**이 발생할 수 있다.

### 1.6.2 전역 변수의 사용을 억제하는 방법

---

전역 변수를 반드시 사용해야 할 이유를 찾지 못한다면 지역 변수를 사용해야한다. 변수의 스코프는 좁을 수록 좋다.

## 1.7 즉시 실행 함수

모든 코드를 즉시 실행 함수로 감싸면 모든 변수는 즉시 실행 함수의 지역 변수가 된다. 이러한 특성을 이용해 전역 변수의 사용을 제한하는 방법

```jsx
(function () {
	var foo = 10;
	...
}())
console.log(foo) // ReferenceError: foo is not defined
```

## 1.8 네임스페이스 객체

전역에 네임스페이스로 사용할 객체를 생성하고 전역변수로 사용하고 싶을 때 변수를 프로퍼티로 추가하는 방식이다.

```jsx
var MYAPP = {}; // 전역 네임스페이스 객체
MYAPP.person = {
  name: "Lee",
  address: "Seoul",
};

console.log(MYAPP.person.name); // Lee
```

## 1.9 모듈 패턴

모듈 패턴은 클래스를 모방해서 관련이 있는 변수와 함수를 모아 즉시 실행 함수로 감싸 하나의 모듈을 만든다. 모듈 패턴의 특징은 **전역 변수의 억제**는 물론 **캡슐화**까지 구현할 수 있다.

`캡슐화`는 객체 상태를 나타내는 프로퍼티와 프로퍼티를 참조하고 조작할 수 있는 동작인 메서드를 하나로 묶는 것을 말한다. `캡슐화`는 객체 특정 프로터피 메서드를 감출 목적으로 사용하기도 하는데 이를 은닉이라 한다.

```jsx
var MyModule = (function () {
  // 프라이빗 변수와 함수
  var privateVar = "I am private";
  function privateFunction() {
    console.log(privateVar);
  }

  // public으로 노출할 프로퍼티와 메서드를 객체 리터럴로 반환
  return {
    publicMethod: function () {
      privateFunction();
    },
    publicVar: "I am public",
  };
})();
console.log(privateVar);
MyModule.publicMethod(); // I am private
console.log(MyModule.publicVar); // I am public
```

## 1.10 ES6 모듈

ES6 모듈을 사용하면 더는 전역 변수를 사용할 수 없다. **ES6 모듈은 파일 자체의 독자적인 모듈 스코프를 제공한다.** 따라서 모듈 내에서 var 키워드로 선언한 변수는 더는 전역 변수가 아니며 window 객체의 프로퍼티도 아니다.

```jsx
<script type="module" src="lib.mjs"></script>
```
